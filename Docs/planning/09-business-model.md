## 1. 개요

세컨드브레인의 **운영 모델**, **역할별 담당자 구조**, **확장 시나리오**를 정의한다.

### 핵심 원칙
1. **개인 자산 우선**: 자신의 지식 축적이 기본
2. **팀 협업 선택**: 필요시 타인과 공유 (Notion/Quartz)
3. **점진적 확장**: MVP → 팀 → 커뮤니티

## 2. MVP 운영 모델 (현재 ~ 3개월)

### 조직 구조
```
┌─────────────────────────────────────┐
│  세컨드브레인 v1.0 (개인 자산)       │
├─────────────────────────────────────┤
│                                     │
│  관리자: 자신 (개발자)              │
│                                     │
│  협력자 (비정규):                   │
│  - 목사님 (기독교 도메인)           │
│  - (추후 추가 가능)                 │
│                                     │
└─────────────────────────────────────┘
```

### 역할 정의

#### 자신 (개발자) — 관리자 & 아키텍트
**책임**:
- 시스템 설계 및 구현 (Phase A~D)
- 운영 및 유지보수
- API 비용 관리
- 도메인별 신뢰 소스 선정 및 평가
- 기술 의사결정

**권한**:
- 모든 범위 결정
- Phase 일정 수정
- 새로운 도메인 추가
- 팀 확대 결정

**투입 시간**: 주 40시간 (MVP 9주) → 주 5시간 (운영)

#### 목사님 — 기독교 도메인 담당
**책임**:
- 기독교 도메인 신뢰 소스 추천
- 설교 주제별 품질 피드백
- 신학 자료 분류 기준 제시

**권한**:
- 기독교 도메인 소스 추가/제거 제안
- 설교 관련 쿼리 우선순위

**투입 시간**: 월 2시간 (협력)
**보상**: 없음 (상호 협력)

## 3. 수익 모델 (현재는 없음)

### 현재 (MVP)
```
비용 구조:
├─ Claude API: $30/월
├─ OpenAI Embedding: $5/월
└─ 인프라 (Docker): $0 (로컬)

수익:
└─ $0 (개인 자산, 비상용)

구조:
비용 → 개인 부담 (소액)
```

### 향후 가능성 (1년+)

#### 시나리오 1: 팀 라이선스 모델 (보수적)
```
대상: 5명 미만 팀 (스타트업, 소규모 조직)

가격: 팀당 $50~100/월
  ├─ 저장소: 공유 Obsidian Vault
  ├─ 사용자: 5명 제한
  ├─ API: 팀 공동 예산
  └─ 지원: 이메일만

수익 모델: 
  5팀 × $75 = $375/월
  → 인프라 + 운영 비용 충당
```

#### 시나리오 2: 도메인 전문 모듈 (중간)
```
대상: 특정 영역 전문가 (신학자, 금융인 등)

상품:
├─ 신학 전문 모듈 (목사님 담당)
├─ 금융 전문 모듈 (재무인 대상)
└─ 기술 전문 모듈 (개발자 대상)

가격: 모듈당 $20~50/월
수익 모델:
  (1000 사용자 × 3 모듈 × $30) = $90K/월
  → 전담 팀 운영 가능
```

#### 시나리오 3: SaaS 플랫폼 (공격적, 2년+)
```
대상: 기업, 대학, 연구기관

기능:
├─ 팀 협업 (다중 사용자)
├─ 권한 관리 (역할 기반)
├─ 고급 분석 (맞춤 보고서)
├─ API 연동 (외부 시스템)
└─ 전담 지원

가격: 조직당 $500~5000/월
수익 모델:
  100 조직 × $2000 = $200K/월
  → 정규 팀 운영 + 개발
```

### 현재 결정
> **MVP 기간(3~6개월): 순수 자산 축적 모드**
> - 수익화 목표 없음
> - 개인 비용으로만 운영
> - 장기 모델은 1년 후 재검토

## 4. 팀 확대 계획 (3~6개월)

### Phase E: Notion 공유 레이어 (3개월 후)

#### 목표
기본적인 팀 협업 가능하게 (비동기 방식)

#### 구성
```
역할 정의:
├─ 관리자 (자신): 전체 Vault 관리
├─ 도메인 담당: 자신의 도메인만 추가/검토
└─ 뷰어 (협력자): 읽기만 가능

Notion 공유 구조:
├─ Database 1: 학술 노트 (개발자만 추가)
├─ Database 2: 기독교 노트 (목사님 읽기)
├─ Database 3: 경제 노트 (대학생 읽기)
└─ 공용: 분석 결과 (모두 볼 수 있음)
```

#### 참여자 추가 기준
| 대상 | 추가 조건 | 권한 |
|------|---------|------|
| 목사님 | MVP 완성 + 설교 피드백 2회 | 도메인 담당 |
| 대학생 | 경제 도메인 요청 + 신뢰 검증 | 도메인 담당 |
| 디자이너 | 트렌드 도메인 구축 | 도메인 담당 |
| 사무원 | 규제 도메인 구축 | 도메인 담당 |

### Phase F: 외부 공개 (6개월 후)

#### 목표
지식 자산 공개 + 포트폴리오/이력서 활용

#### Quartz 배포 전략
```
공개 범위:
├─ ✓ 공개: 일반적 지식 (영구 노트, 학술 논문)
├─ ⚠️ 제한: 개인 프로젝트 (직업 정보, 취업 전략)
└─ ✗ 비공개: 민감 정보 (개인 상담 기록, 재무)

필터링:
1. Frontmatter에 `public: true` 추가된 노트만
2. Quartz 빌드시 필터링
3. SEO 최적화 (검색 노출)
```

#### 예상 효과
- **개인 브랜딩**: "세컨드브레인으로 지식 관리하는 개발자"
- **취업 가산점**: 포트폴리오로 활용 가능
- **커뮤니티 기여**: 공유된 지식으로 피드백 수집

## 5. 팀 규모별 운영 계획

### 1. MVP 단계 (지금 ~ 3개월)
```
팀 규모: 1명 (자신)
목표: 자동 수집 + 기본 분석 작동

운영 비용: $35/월 (자신 부담)
의사결정: 자신이 전담
점검: 주 1회 자체 검증
```

### 2. 소규모 팀 단계 (3~6개월)
```
팀 규모: 2~5명
  - 자신 (아키텍트)
  - 목사님 (기독교)
  - (추가 2~3명)

목표: Notion 공유 + 팀 협업

운영 비용:
  - 기술: $35/월 (자신 부담)
  - 협력자: 무급 (상호 협력)
  - 총: $35/월

의사결정:
  - 기술: 자신
  - 도메인: 각 담당자
  - 분쟁: 자신이 조정

점검: 월 1회 팀 회의
```

### 3. 중규모 팀 단계 (6~12개월)
```
팀 규모: 5~10명
  - 자신 (리더)
  - 도메인 담당자 4명
  - 운영 지원 1명

목표: Quartz 공개 + 정기적 분석 보고서

운영 비용:
  - 기술: $100~200/월
  - 협력자 보상: $0 (여전히 자발적)
  - 클라우드: $200~500/월 (검토)
  - 총: $300~700/월

의사결정:
  - 분기 팀 회의 (공식 의사결정)
  - 도메인 소위원회 (자체 결정)

점검: 월 1회 + 분기 리뷰
```

### 4. 조직 단계 (1년+, 검토 중)
```
팀 규모: 10명↑ 또는 다중 팀

목표: SaaS 플랫폼 검토 / 다중 보금자리

운영 모델: 미정 (1년 후 재검토)
```

## 6. 도메인별 담당자 모델

### 도메인 구조
```
세컨드브레인
├─ 기독교 도메인
│  └─ 담당: 목사님
│     ├─ 신뢰 소스 선정
│     ├─ 품질 검수
│     ├─ 공유 대상 결정
│     └─ 분석 요청 우선순위
│
├─ 학술 도메인
│  └─ 담당: 개발자
│     └─ (위와 동일)
│
├─ 경제 도메인
│  └─ 담당: 대학생 + 사무원 (공동)
│     └─ (위와 동일)
│
└─ 디자인 도메인 (향후)
   └─ 담당: 디자이너
      └─ (위와 동일)
```

### 담당자 책임 및 권한

#### 책임
1. **소스 관리**: 신뢰 소스 5~10개 선정 및 평가
2. **품질 검수**: 월 1회 생성 노트 샘플 검토
3. **피드백 제공**: 분석 결과 유용성 평가
4. **도메인 진화**: 3개월마다 소스 추가/제거 제안

#### 권한
1. **소스 추가**: 담당 도메인 신뢰 소스 추가 제안
2. **우선순위**: 분석 요청시 담당 도메인 우선 처리
3. **메타데이터**: 도메인별 태그/카테고리 정의
4. **공유 대상**: 도메인 노트의 공개 범위 결정

#### 투입 시간
- **초기 (1개월)**: 10시간 (소스 선정 + 가이드 작성)
- **운영 (매월)**: 2시간 (검수 + 피드백)
- **분기 리뷰**: 2시간 (개선 사항 논의)

#### 보상 (현재)
- **무급**: 상호 협력 기반
- **향후**: 조직 확대시 검토

## 7. 협업 워크플로우

### 월간 리듬

#### Week 1: 수집 & 검수
```
자신:
  - n8n 자동 수집 확인
  - 이번 주 노트 20개 생성 확인
  - 도메인 담당자에게 샘플 전송

도메인 담당자:
  - 할당받은 샘플 노트 5개 검토
  - "좋음", "개선 필요" 피드백
  - 이메일/Slack으로 반영
```

#### Week 2: 분석 & 피드백
```
자신:
  - 담당자 피드백 반영
  - 프롬프트 개선 (필요시)
  - 새 도메인 소스 제안 검토

사용자:
  - 분석 요청 (필요시)
  - "이 분석 유용했나?" 피드백
```

#### Week 3: 통합 & 개선
```
자신:
  - 수집 데이터 품질 분석
  - API 비용 점검
  - 기술 개선사항 정리
```

#### Week 4: 리뷰 & 계획
```
분기별 (Week 4의 월 4주차):
  - 팀 회의 (온라인)
  - 지난 분기 결과 리뷰
  - 다음 분기 계획 수립
  - 도메인별 로드맵 갱신
```

## 8. 성장 시나리오

```
Timeline: 12개월

  - 도메인: 5개

```
Timeline: 18개월
- Month 1~3: MVP (개인)
- Month 4~6: 목사님 협력 (기독교)
- Month 7~9: 대학생 추가 (경제)

Result:
  - 노트: 300~500개

  - 도메인: 3개 (안정)
  - 월간 분석: 10~20건

```
Timeline: 24개월
- Phase 1~2 (Month 1~6): MVP
- Phase 3~4 (Month 7~12): 팀 협력 + 웹 UI
- Phase 5 (Month 13~18): 기업 고객 1~2개 테스트
- Phase 6 (Month 19~24): 정식 SaaS 출시

Investment:
  - 개발: 400시간 (개발자 6개월)
  - 인프라: $2000~5000 (클라우드)
  - 마케팅: $1000~3000

Risk:
  - 시장 수요 불확실
  - 경쟁 증가
  - 운영 난제
```

## 9. 재정 계획

### 비용 구조

#### Fixed Cost (월)
| 항목 | 비용 | 설명 |
|------|------|------|
| Claude API | $30 | 문헌 노트 생성 + 분석 |
| OpenAI Embedding | $5 | 벡터 임베딩 |
| **월간 고정** | **$35** | |

#### Variable Cost
| 항목 | 범위 | 조건 |
|------|------|------|
| 추가 분석 | $0~20 | 질문이 많을 때 |
| 클라우드 (향후) | $0~500 | 배포시 |
| **월간 변동** | **$0~20** | |

#### 연간 예상
```
Year 1: $35 × 12 = $420 (MVP)
Year 2: $50 × 12 = $600 (팀 협력)
Year 3: $100~500 × 12 = $1200~6000 (클라우드 검토)
```

### 수익 계획 (선택)

#### Year 1: 수익 $0
- 개인 자산 축적 기간
- 모든 비용 자신이 부담

#### Year 2: 검토 중
- 팀 협력 기반 조성
- 소규모 라이선스 모델 검토

#### Year 3+: 선택적
- SaaS 플랫폼화 (큰 위험)
- 또는 계속 개인 자산 (현재 방향)
