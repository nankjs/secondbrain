## 1. 개요

이 문서는 세컨드브레인 플랫폼의 **핵심 사용 시나리오**를 담당 역할별로 기술한다. 각 시나리오는 실제 사용자가 겪는 의사결정 순간에서 시작되며, 세컨드브레인이 어떻게 문제를 해결하는지 단계별로 설명한다.

---

## 2. 시나리오 1: 개발자 — 새 기술 스택 선택

### 상황
팀에서 데이터 처리 파이프라인을 구축해야 하는데, Python 데이터 과학 라이브러리로 **pandas vs Polars vs DuckDB** 중 선택해야 한다. 2주 안에 최종 결정을 내려야 하고, 기술 리더에게 근거 있는 제안을 해야 한다.

### 문제점 (Without 세컨드브레인)
- 매번 구글, GitHub Trending, Reddit에서 검색
- 지난달에 읽은 좋은 벤치마크 글을 못 찾음
- 팀 내 기술 선택 이력 분산 저장

### Phase별 사용 흐름

#### Phase A~B (기초 수집)
**시간**: 지난 3개월, 백그라운드 자동 진행

| 시점 | 액션 | 상태 |
|------|------|------|
| 2월 1주 | n8n이 ArXiv에서 "data processing Python"으로 검색 | 5개 논문 수집 |
| 2월 3주 | GitHub Trending에서 "data-science" 리포 모니터링 | 10개 리포 추가 |
| 3월 초 | 기술 뉴스 RSS에서 pandas/polars 비교 글 자동 수집 | 문헌 노트 생성 |
| 3월 말 | Obsidian에 30개 관련 노트 축적 (자동 분류) | 준비 완료 |

**Phase A 결과**: 3개월 동안 관련 자료 30개 축적
- 01-literature/: 논문, 기사, 리포지토리 정보
- 02-permanent/: "데이터 처리 라이브러리 비교" 영구 노트 생성

#### Phase C (지식 연결)
**AI가 자동으로 연결**: 
- "Polars vs Pandas 성능 비교" ← "데이터 처리 라이브러리 비교" 관련도 95%
- "DuckDB 쿼리 최적화" ← "컬럼 지향 DB" 관련도 88%

#### Phase D (의사결정 직전, 오늘)
**시간**: 의사결정 2시간 전

```
1. Obsidian 열기
   └─ 검색어: "Python 데이터 처리 라이브러리 비교"

2. 세컨드브레인에 질문
   └─ "2026년 pandas vs Polars vs DuckDB 선택 기준?"

3. AI 분석 (Claude MCP + LightRAG)
   └─ 1분: 벡터 검색 (관련 노트 20개 추출)
   └─ 2분: 보고서 생성 (근거별 정렬)
   └─ 1분: 신뢰성 검수

4. 결과 (5분 내 받음)
   ├─ 요약: "2026년 기준, 성능은 Polars > DuckDB > pandas"
   ├─ 근거:
   │  ├─ ArXiv 논문 (2024-Q4): Polars 처리 속도 10배 빠름
   │  ├─ GitHub Trending: Polars 스타 증가율 최고
   │  └─ 커뮤니티 의견: 프로토타입용 DuckDB, 프로덕션용 Polars
   └─ 참고: [[Polars vs Pandas]], [[DuckDB 최적화]]

5. 의사결정
   └─ "팀에 Polars 제안" (근거 3개 제시)
```

### 성과
- **시간 절감**: 일일이 검색하던 2시간 → 5분
- **근거 확보**: 최근 논문 + 커뮤니티 의견 + 기술 트렌드 종합
- **일관성**: 팀 기술 결정 이력 Obsidian에 기록 (향후 참고 가능)

### 피드백 루프
개발자가 선택 후 3개월 뒤 실제 사용 결과를 새로운 노트로 기록
→ AI가 학습해서 다음 선택 시 더 정확한 제안 (선택 이유 가중치 조정)

## 3. 시나리오 2: 목사님 — 설교 자료 준비

### 상황
주일 설교 주제: "부모 자식 관계의 성경적 기초"

일반적으로 설교 준비에:
- 신학 자료 찾기: 1시간 이상
- 최근 신앙 뉴스/사건 연결: 30분
- 설교 구성 (기존 설교 참고): 1시간
- **총: 3시간 이상**

### 문제점 (Without 세컨드브레인)
- 신학 자료 산재 (도서, PDF, 웹페이지)
- 과거 설교 자료 어디 저장했는지 못 찾음
- 최근 신앙 관련 뉴스 놓침 (자동 수집 불가)

### Phase별 사용 흐름

#### Phase A~B (백그라운드)
**3개월 동안 자동 수집**:
- 신학 RSS: 복음주의신학회, 개혁신학지 등 → 주 10개 자료
- 신앙 뉴스: 기독일보, 뉴스앤조이 → 주 15개 기사
- 목사님 추천 영상/책: 수동으로 추가

**Phase A 결과**: Obsidian에 기독교 관련 노트 150개 축적

#### Phase C (지식 연결)
자동 연결:
- "부자 관계 신학" ← "창조와 의존 관계" (관련도 92%)
- "에베소서 6:1-9" ← "자녀 교육 성경" (관련도 96%)

#### Phase D (설교 당일 아침, 1시간 전)
**시간**: 최종 점검

```
1. Obsidian 열기
   └─ 태그: #설교주제/부모자식

2. 세컨드브레인 질문
   └─ "부모 자식 관계의 성경적 기초 - 2026년 신앙 트렌드 포함"

3. AI 분석 (3분)
   ├─ 신학 자료 (영구 노트): 5개
   ├─ 최근 설교 (문헌 노트): 3개
   └─ 신앙 뉴스: 2개

4. 결과
   ├─ 신학적 배경
   │  ├─ 에베소서 6:1-9 (자녀 순종)
   │  ├─ 출애굽기 20:12 (부모 공경)
   │  └─ 현대 신학: 독립성과 관계의 균형
   ├─ 설교 구성 안
   │  ├─ 도입: 최근 신앙 뉴스 사례 (가정 위기)
   │  ├─ 전개: 성경 기초 (3가지 핵심 진리)
   │  └─ 결론: 행동 과제 (기도 + 관계 회복)
   └─ 참고: [[에베소서]], [[가족윤리]], [[현대가정문제]]

5. 설교 준비
   └─ 기존 설교문보다 30분 단축
       (자료 찾는 시간 줄고 구성에만 집중)
```

### 성과
- **시간 절감**: 3시간 → 1시간 30분 (50% 단축)
- **신학적 깊이**: 신학 자료 + 실시간 신앙 뉴스 통합
- **설교 질 향상**: 근거 있고 시의성 있는 메시지

### 피드백 루프
설교 후 시간이 나면 "이번 설교에서 잘된 점" 노트 기록
→ 향후 유사 주제 설교 시 참고 → AI가 학습해서 더 빠른 제안

## 4. 시나리오 3: 대학생 — 취업 준비

### 상황
3학년, 9개월 뒤 취업 면접 준비

필요한 정보:
- 업계 기초 지식 (금융, IT 중 선택)
- 기업 분석 자료
- 면접 자주 나오는 질문 + 모범 답변
- 경제 동향 (면접에서 시사 질문 대비)

### 문제점 (Without 세컨드브레인)
- 찾은 정보 파편화 (유튜브, 블로그, 논문 혼재)
- 노트북/폴더 정리 복잡
- 지난달 찾은 "좋은 기업 분석" 못 찾음
- 경제 뉴스를 매일 읽지 않으면 정보 놓침

### Phase별 사용 흐름

#### Phase A~B (장기 축적, 9개월)
**월별 진행**:

**1~2개월**:
- 금융 학과 필수 자료 (신용시스템, 자산관리 기초)
- 금융 뉴스 RSS 구독 시작
- 금융권 ETF 추적 (yfinance)

**3~4개월**:
- ArXiv: "금융 머신러닝", "위험 관리" 논문 수집
- 기업 분석 자료 (삼성전자, 한국은행 뉴스) 축적
- 면접 기출 질문 노트 시작

**5~6개월**:
- 경제 트렌드 (금리, 환율, 부동산) 연결
- 역사와의 맥락 (2008년 금융위기 vs 2024년 시장 상황)

**7~8개월**:
- 면접 예상 질문 정리 (지난 5년 금융사 면접 + 최신 경제 이슈)
- 모범 답변 템플릿 작성

**9개월** (면접 1개월 전):
- 최종 정리 완료
- 150개 이상 관련 노트 축적

#### Phase C (지식 연결)
AI가 자동으로:
- "2024년 금리 인상" ← "중앙은행 통화정책" (관련도 98%)
- "부동산 시장 냉각" ← "경제 사이클" (관련도 85%)
- "삼성 반도체 경쟁력" ← "글로벌 칩 수급" (관련도 92%)

#### Phase D (면접 1주일 전)

```
1. 최종 면접 대비
   └─ 질문: "한국은행 경영진 면접 - 최근 3개월 경제 이슈"

2. AI 보고서 (5분)
   ├─ 면접 예상 질문 5개 + 모범 답변
   ├─ 최근 경제 뉴스 정리 (금리, 환율, 부동산)
   ├─ 한국은행 관련 보도 5개 (지난 3개월)
   └─ 유사 기업 경쟁사 분석

3. 면접 현장
   ├─ 예상 질문 "요즘 금리를 어떻게 보세요?"
   │  └─ 대답: "3개월간 추적한 데이터에 따르면..."
   │     (세컨드브레인에서 정리한 근거 활용)
   └─ 시사 질문도 자신감 있게 답변 가능

4. 면접 결과
   └─ 합격
```

### 성과
- **학습 효율**: 9개월 동안 체계적 축적 (한 달씩 구글링이 아님)
- **면접 대비**: 최신 정보 + 이론 통합
- **장기 자산**: 향후 금융 관련 공부할 때도 활용 가능

## 5. 시나리오 4: 웹디자이너 — 디자인 트렌드 추적

### 상황
프리랜서 웹디자이너, 클라이언트마다 트렌디한 UI/UX 제안 필요

필요:
- 월별 디자인 트렌드
- 웹사이트 포트폴리오 (UI 조금 "차용")
- 컬러, 타이포그래피 최신 동향
- 사용성 연구 (UX 근거)

### 문제점 (Without 세컨드브레인)
- Pinterest, Dribbble에서 매번 스크롤
- 발견한 좋은 디자인 저장했는데 정렬 안 됨
- 클라이언트마다 처음부터 리서치 시작

### Phase별 사용 흐름

#### Phase A~B (디자인 트렌드 수집)
**자동 수집**:
- Design News RSS (Adobe, Dribbble, Smashing Magazine)
- GitHub: 오픈소스 UI 프레임워크 (Shadcn, Mantine)
- ArXiv: "HCI 연구", "사용성 연구" 논문

#### Phase C (연결)
- "Dark Mode 트렌드" ← "접근성 (Accessibility)" (관련도 89%)
- "Glassmorphism" ← "뉴모피즘" (관련도 78%, 진화 관계)

#### Phase D (프로젝트 시작 시)

```
1. 새 클라이언트 프로젝트: 금융앱 UI
   └─ 질문: "2026년 금융앱 UI/UX 트렌드 + 사용성 근거"

2. AI 보고서 (5분)
   ├─ 디자인 트렌드: 미니멀리즘 + 3D 요소
   ├─ 컬러: 신뢰감 있는 블루/그린 톤
   ├─ 타이포: Sans-serif 기본, 숫자는 Monospace
   ├─ 사용성: 터치 타겟 48px 이상 (근거: Nielsen Norman)
   └─ 포트폴리오: 관련 사례 3개 링크

3. 디자인 시안
   └─ 트렌드 + 근거 있는 제안 (클라이언트 신뢰 향상)

4. 결과
   └─ 프로젝트 리드 타임 30% 단축
```

## 6. 시나리오 5: 사무원 — 법규/공지 모니터링

### 상황
회사 법무팀, 월 30건 이상 새로운 규제/공지 추적

필요:
- 산업 관련 법규 변화
- 정부 공시 (금융감독청, 공정거래위원회)
- 회사 관련 뉴스 모니터링

### 문제점 (Without 세컨드브레인)
- 여러 웹사이트 매일 방문 (1시간/일)
- 놓친 공지 있을 수 있음
- 과거 사건과의 관련성 파악 어려움

### Phase별 사용 흐름

#### Phase A~B (법규 자동 모니터링)
**n8n 자동화**:
- RSS: 금융감독청, 공정거래위원회, 관보
- API: 정부 공지 자동 모니터링
- 뉴스: 회사명 관련 기사 자동 수집

**월별**: 100개 이상 새로운 법규/공지 축적

#### Phase C (맥락 연결)
- "2026년 금융감시 강화" ← "과거 금융위기" (역사적 맥락)
- "공정거래법 개정안" ← "대기업 내부 거래" (관련 판례)

#### Phase D (월간 보고)

```
1. 월말 리뷰
   └─ 질문: "이달 금융업 규제 변화 - 우리 회사 영향도"

2. AI 보고서 (5분)
   ├─ 새로운 법규 5개 (영향도 정렬)
   ├─ 각 규제별 우리 회사 체크리스트
   ├─ 과거 사건과의 연결 (학습 포인트)
   └─ 담당 부서별 액션 아이템

3. 경영진 보고
   └─ 법무팀이 정리한 리포트 (근거 있음)

4. 성과
   ├─ 수동 모니터링 시간 1시간/일 → 0.5시간/일 (50% 단축)
   └─ 놓친 공지 0건 (자동화)
```

## 7. 교차 시나리오: 팀 협업 (Phase E~F)

### 상황 (6개월 뒤)
개발자, 목사님, 대학생, 디자이너, 사무원 5명이 Notion으로 공유 시작

**팀 워크플로우**:
```
Obsidian Vault (개발자 관리)
    ↓
[Python 동기화]
    ↓
Notion Database (역할별 공유)
    ├─ 관리자 (개발자): 전체
    ├─ 도메인 담당자: 자신의 도메인만
    └─ 뷰어: 요약 & 링크만
    ↓
Quartz (선택): 외부 공개
```

### 협업 시나리오
**회의**: "2026년 상반기 금융 시장 전망"

참석자: 개발자 + 사무원 + 목사님(선택)

```
1. 개발자 질문
   └─ "2026년 상반기 금융 시장 - AI/기술 영향도"

2. AI 보고서 (5분)
   ├─ 기술 관점: AI 금융 서비스 확장
   ├─ 규제 관점: 금융감독 강화 (사무원 수집 자료 포함)
   ├─ 신앙 관점: 경제 정의 (목사님 설교 자료 포함)
   └─ 통합 분석

3. 회의
   └─ 다양한 관점 + 근거로 깊이 있는 토론

4. 합의
   └─ "상반기 투자 전략 수립"
```

### 성과
- **크로스펑셔널**: 서로 다른 도메인 정보 통합
- **신뢰성**: 각자 축적한 근거 자료 활용
- **효율성**: 회의 준비 시간 대폭 단축

## 9. 사용성 원칙

### 설계 원칙
1. **자동 → 수동**: 백그라운드 자동화 우선, 필요할 때만 수동 입력
2. **검색 → 분석**: 원문 검색 X, AI 분석으로 직결
3. **분산 → 중앙**: 파편화된 자료 → Obsidian Vault 단일 저장소
4. **당장 → 미래**: 현재 필요뿐 아니라 미래 활용 가능한 형식

### UX 원칙
- **최소 클릭**: 질문 → 결과 3단계 이내
- **명확한 근거**: 모든 결론에 출처 명시 (노트 링크)
- **선택적 상세화**: 기본은 요약, 원하면 상세 자료 연결
- **피드백 루프**: 사용자 수정 학습 (AI 품질 향상)

## 10. 예상 문제 및 대응

### 문제 1: 수집 자료 품질 편차
**증상**: 신뢰할 수 없는 정보 섞임

**대응**:
- Phase B: 신뢰 소스 엄선으로 시작 (편집부 판단)
- Phase C 이후: AI가 "신뢰도 점수" 자동 매김
- 월 1회: 사용자 피드백으로 소스 재평가

### 문제 2: AI 분석 정확도 저하
**증상**: "이건 내 질문과 상관없는데?"

**대응**:
- 벡터 검색 임계값 조정 (현재 0.7 → 0.8)
- 사용자 수정 피드백 누적으로 학습
- 분기별 품질 검수

### 문제 3: 정보 과부하
**증상**: 너무 많은 노트, 연결 파악 어려움

**대응**:
- Obsidian Graph View로 시각화
- 월 1회 아카이빙 (1년 이상 미사용 → 07-archive/)
- 집중 영역 필터링 (태그로 선택적 보기)
