## 1. 개요

### 프로젝트명
**아바타 프로젝트 / 세컨드브레인** (Avatar Project / SecondBrain)

### 한 줄 설명
사용자의 디지털 분신 — 24/7 자동으로 신뢰 정보를 수집하고, 의사결정 직전에 근거 자료를 제공하는 지식 자동화 플랫폼

### 비전
개인과 팀이 정보 파편화에서 벗어나, **빠른 의사결정**과 **장기 자산 축적**을 동시에 달성하는 정보 생태계 구축

---

## 2. 문제 정의

### 사용자가 겪는 주요 문제

| 문제 | 영향 |
|------|------|
| **수집 소스 불명확** | 신뢰할 수 있는 정보 찾는 데 시간 낭비 |
| **자료 파편화** | 흩어진 정보 — 저장해도 못 찾거나 맥락 소실 |
| **재활용 불가** | 과거 검색 자료 재활용 불가능, 매번 처음부터 시작 |
| **미래 자료 포기** | 당장 필요한 것만 검색 → 축적 불가 |
| **의사결정 시간 부족** | 중요한 결정 앞에서 근거 자료 부족 |

### 영향받는 역할
- **개발자**: 새로운 기술 스택 선택, 라이브러리 벤치마크 필요
- **목사님**: 설교 주제별 신학 자료, 신앙 관련 뉴스 수집
- **대학생**: 취업 준비, 학과 관련 기초 지식 축적
- **웹디자이너**: 디자인 트렌드, UI/UX 사례 모음
- **사무원**: 업무 관련 법규, 공지사항, 프로세스 자료

---

## 3. 해결책: 세컨드브레인

### 핵심 가치 제안
1. **24/7 자동 수집** — 사용자가 지정한 신뢰 소스에서 자동으로 정보 모음
2. **스마트 연결** — AI가 새로운 자료를 기존 지식과 자동 연결
3. **즉시 분석** — 한 줄 질문으로 5분 안에 근거 있는 보고서 생성

### 차별화 요소
- **신뢰성 우선**: AI 학습보다 소스 엄선으로 시작
- **맥락 보존**: Zettelkasten 기반 지식 그래프로 자료 간 관계 유지
- **온디맨드 분석**: 필요할 때만 검색·요약 (낭비 최소화)
- **역할별 분리**: 팀 확장 시 권한 관리 가능

## 4. 핵심 기능 (MVP)

### 기능 1: 자동 수집 (Collection)
**목적**: 신뢰 소스에서 24/7 자동으로 새 정보 수집

**범위 (Phase B)**:
- 3개 도메인 모니터링
  - 기독교 자료 (신학, 설교, 신앙 뉴스)
  - 학술 자료 (ArXiv 논문, 학위논문)
  - 경제·주식 데이터 (국내 ETF, 암호화폐 기초)
- 수집 빈도: 주 1회 자동 실행
- 도메인별 10~20개 신뢰 소스 선정

**기술**:
- n8n 스케줄 + RSS, ArXiv API, yfinance
- Claude 스킬 `/literature` (원자화)

**성공 기준**:
- 주 1회 실행 시 매번 새 노트 5개 이상 자동 생성
- 0% 오류율 (예약 실행 100% 완료)

---

### 기능 2: 지식 연결 (Knowledge Linking)
**목적**: 새로운 자료가 기존 지식과 자동으로 연결되는 관계도 구축

**범위 (Phase C)**:
- 수집한 노트와 기존 노트 간 자동 연결
- AI 추천 연결 기본, 사용자 수정 학습으로 품질 향상
- Zettelkasten 구조: 떠돌이(Fleeting) → 문헌(Literature) → 영구(Permanent) 노트로 진화

**기술**:
- Zettelkasten MCP + Claude 스킬 (`/permanent`, `/lint`, `/index`)
- 임베딩 기반 유사도 검색 (LightRAG)
- Obsidian Vault 저장소

**성공 기준**:
- 3개월 내 연결된 노트 100개 이상 축적
- 수동 연결 비율 < 20% (자동 연결 > 80% 정확도)

---

### 기능 3: 즉시 분석 (Quick Analysis)
**목적**: 한 줄 질문으로 5분 안에 신뢰할 수 있는 근거 보고서 생성

**범위 (Phase D)**:
- 사용자 질문 수신 → 관련 노트 자동 검색 → AI 보고서 생성
- 형식: 마크다운 헤드 요약 (기본), 선택 시 슬라이드/표 추가
- 응답 시간: 5분 이내 (실시간 선호)

**기술**:
- LightRAG Docker (질문→노트 검색)
- GPT Researcher (보고서 자동 생성)
- Claude MCP (온디맨드 분석)

**성공 기준**:
- 쿼리당 응답 시간 < 5분
- 보고서 유용성 자가평가 4/5 이상 (3주 연속)

## 5. 사용자 및 사용 맥락

### 주요 사용자
1. **개발자 (나)** — 기술 스택 선택, 라이브러리 벤치마크, 면접 대비
2. **목사님** — 설교 자료, 신학 이해, 신앙 뉴스 종합
3. **대학생** — 취업 준비, 학과 전공 심화
4. **웹디자이너** — 디자인 트렌드, UI/UX 모범 사례
5. **사무원** — 법규 참고, 공지 모니터링, 프로세스 자료

### 사용 시나리오
**시나리오 1**: 회사에서 새 프로젝트 기술 선택 직전
- 개발자가 세컨드브레인에 "Python 데이터 과학 라이브러리 최신 동향"이라고 질문
- 5분 내 지난 3개월 ArXiv 논문 + GitHub Trending 기반 요약 보고서 수신
- 의사결정 근거 확보 후 기술 선택

**시나리오 2**: 설교 주제 준비 (목사님)
- "부모 자식 관계의 성경적 기초"로 검색
- 과거 3년 수집한 신학 자료 + 최신 설교 사례 종합 보고서
- 설교 구성 1시간 단축

## 6. 성공 지표 (KPI)

### 정량 지표
| 지표 | 목표 | 측정 주기 |
|------|------|---------|
| 자동 수집 신규 노트 | 주 5개 이상 | 주 1회 |
| 누적 연결 노트 | 3개월 100개 | 월 1회 |
| 질문 응답 시간 | < 5분 | 실시간 |
| 사용자 만족도 | 4/5 이상 | 주 1회 자가평가 |

### 정성 지표
- "이 정보가 없었으면 실수했을뻔"이라는 경험 월 1회 이상
- 과거 수집 자료 재활용 빈도 증가
- 팀 규모 확장 시 다른 역할 참여 의향

## 7. 범위 (Scope)

### 포함 범위
- Obsidian Vault 기반 개인 지식 관리
- 3개 도메인 자동 수집 (기독교, 학술, 경제)
- AI 기반 자동 연결 및 분석
- 역할별 담당자 모델 (향후 팀 확장 가능)
- Docker 기반 로컬 배포

### 제외 범위 (MVP)
- 모바일 앱 (Web UI만 지원)
- 소셜 자동 발행 (Threads, X)
- 실시간 협업 기능 (비동기 피드백만)
- Quartz 외부 공개 (Phase F, 마지막 우선순위)
- 동시 다사용자 하이브리드 구조

## 9. 의존성 및 전제조건

### 기술 환경
- Windows 11 PC (24/7 가동 가정)
- Docker Desktop 설치
- Python 3.11+
- Node.js 18+
- Obsidian 신규 볼트

### 리소스
- 구현 담당: 1명 (나)
- 검토자: 필요시 역할별 담당자 (목사님, 팀원)
- 신뢰 소스 선정: 역할별 담당자 협력

### 외부 의존성
- n8n 클라우드 (또는 로컬 설치)
- ArXiv, yfinance API (무료)
- Claude API (비용 발생)
- LightRAG 오픈소스
