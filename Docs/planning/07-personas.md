## 1. 개요

세컨드브레인의 5가지 핵심 사용자 페르소나를 정의한다.

각 페르소나는:
- **역할**: 실제 역할 (직책, 전문 영역)
- **목표**: 해결하려는 문제 (JTBD)
- **행동**: 세컨드브레인 사용 방식
- **성공 지표**: 각자에게 중요한 KPI
- **참여 타이밍**: MVP 및 확장 단계별 참여 시점

## 2. 페르소나 1: 개발자 (나)

### 프로필
| 항목 | 내용 |
|------|------|
| 이름 | 자신 (개발자) |
| 나이 | 30대 |
| 직책 | 프리랜서 / 취업 준비 |
| 경험 | 5년 이상 개발 경험 |
| 기술 수준 | 고급 |
| 사용 기기 | Windows 11 PC + 노트북 |

### 역할 및 책임
- 세컨드브레인 **설계자 및 유지보수자**
- 기술 의사결정 담당
- 시스템 아키텍처 최적화

### 문제점 (JTBD: Jobs To Be Done)

**당장 해야 할 일 (Immediate)**:
- 새로운 기술 스택 선택 시 신뢰할 수 있는 근거 필요
  - "Pandas vs Polars, 어느 걸 쓸까?"
  - "새 프레임워크 도입해도 되나?"
- 지난 3개월간 본 논문/기사 중 관련 자료를 못 찾음
  - "어디서 읽었는데..." → 검색 불가

**장기적 필요 (Long-term)**:
- 기술 경향 트래킹 (자동화로 시간 절약)
- 개인 지식 자산 구축 (취업 면접/포트폴리오용)
- 팀 기술 결정 이력 기록 (재참고 가능)

### 세컨드브레인 사용 방식

#### Daily Workflow
```
아침 (07:00):
  1. Slack 알림 확인: "지난밤 8개 자료 수집됨"
  2. Obsidian 빠른 훑어보기 (2분)
  3. 관심 영역 최신 정보 확인

오후 (14:00, 필요시):
  1. 질문: "2026 Python ML 라이브러리 비교"
  2. 5분 대기
  3. 분석 결과 수신 (요약 + 근거 자료)
  4. 팀에 제안

저녁 (19:00, 선택):
  1. 좋은 논문/기사 발견시 수동 저장
  2. "이 연결 맞음" 피드백
  3. 월 1회 새로운 도메인 추가 검토
```

#### 중요한 순간
**기술 선택 직전** (가장 중요):
- "우리 팀이 이 기술을 써도 되나?"
- 최신 정보 + 커뮤니티 의견 + 성숙도 판단
- 의사결정 시간: 2시간 → 30분 단축 목표

**면접/포트폴리오 준비**:
- "내 전공 기술의 최신 동향은?"
- 축적된 자료로 깊이 있는 답변 가능

### 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 일일 사용 | 1회 이상 | Obsidian 사용 로그 |
| 의사결정 시간 | 50% 단축 | 자가 평가 |
| 재활용 비율 | 80% 이상 | 3개월 뒤 조사 |
| 기술 선택 정확도 | "맞게 선택했다"는 경험 월 1회 | 자가 평가 |
| 축적 노트 | 월 50개 | 통계 |

### 참여 타이밍

| Phase | 역할 | 활동 |
|-------|------|------|
| Phase A~F | 설계 & 구현 | 모든 Phase 주도 |
| MVP 완성 후 | 주요 사용자 | 일일 사용, 피드백 |
| 6개월 후 | 팀 리더 | 목사님/대학생 온보딩 |

## 3. 페르소나 2: 목사님

### 프로필
| 항목 | 내용 |
|------|------|
| 이름 | 목사님 |
| 나이 | 50대 |
| 직책 | 담임목사 |
| 경험 | 20년 이상 사역 경험 |
| 기술 수준 | 중급 (스마트폰, 기본 PC) |
| 사용 기기 | iPhone + Windows PC (가끔) |

### 역할 및 책임
- 주일 설교 준비
- 성경 공부 인도
- 신앙 상담

### 문제점 (JTBD)

**당장 해야 할 일**:
- 주일 설교 준비 시간 3시간 → 1시간으로 단축
  - "부모 자식 관계의 성경적 기초"
  - 신학 자료 + 최근 뉴스 사례 + 과거 설교 참고
- 설교 주제에 맞는 신학 자료를 매번 찾음
  - "이전에 정리한 자료 어디 갔더라?"

**장기적 필요**:
- 설교 주제별 지식 축적 (향후 참고용)
- 신앙 뉴스 자동 모니터링 (시의성 있는 설교)
- 교인들과 설교 요약 공유 (Notion 활용)

### 세컨드브레인 사용 방식

#### Weekly Workflow
```
주 초 (월요일):
  1. 설교 주제 결정 (기도 후)
  2. 개발자에게 "다음 주 주제 입력" 메시지
  3. AI가 자동으로 관련 자료 모음

주 중 (수요일):
  1. 세컨드브레인에 축적된 자료 검토
  2. 개발자가 정리한 분석 보고서 읽음
  3. 설교 초안 작성 (1시간)

주말 (금토):
  1. 설교 예습 및 기도
  2. 최종 점검
```

#### 중요한 순간
**설교 준비 직전** (가장 중요):
- "이 주제에 대한 근거 자료는?"
- 신학적 기초 + 시사성 (최근 신앙 뉴스)
- 준비 시간 3시간 → 1시간 단축

**교인 상담 중**:
- "성경에서는 이 문제에 대해 뭐라고 하나?"
- 빠르게 관련 경전 및 신학 자료 참고

### 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 설교 준비 시간 | 50% 단축 (3h → 1.5h) | 자가 평가 |
| 설교 품질 | 근거 있는 메시지 | 자가 평가 |
| 신앙 뉴스 수집 | 매주 3건 이상 | 세컨드브레인 자동 |
| 재사용 빈도 | 과거 자료 월 2회 이상 | 로그 |
| 교인 피드백 | "깊이 있는 설교" 의견 월 1회 | 피드백 |

### 참여 타이밍

| Phase | 역할 | 활동 |
|-------|------|------|
| Phase B | 신뢰 소스 제공 | 기독교 뉴스 소스 5개 추천 |
| Phase D | 초기 사용자 | 실제 설교 주제로 테스트 |
| 6개월 후 | 정기 사용자 | 주간 설교 준비 자동화 |
| 1년 후 | 팀 멤버 | 교인 공유용 Notion 활용 |

### 특수 고려사항
- **기술 이해도**: 낮음 → 사용 방식 최대한 단순화
- **사용 장치**: 모바일 중심 (향후 Mobile Web UI 필요)
- **시간 한계**: 설교 준비시에만 사용 → 주간 스케줄 맞춤

## 4. 페르소나 3: 대학생 (취업 준비)

### 프로필
| 항목 | 내용 |
|------|------|
| 이름 | 대학생 (3학년) |
| 나이 | 20대 |
| 전공 | 금융/경제학 |
| 경험 | 0~2년 (신입 준비) |
| 기술 수준 | 중급 (스마트폰, PC 기본) |
| 사용 기기 | 맥북 + 아이폰 |

### 역할 및 책임
- 취업 준비 (6개월 집중)
- 기업 분석 및 면접 준비
- 포트폴리오 구축

### 문제점 (JTBD)

**당장 해야 할 일**:
- 업계 기초 지식 축적 (면접 대비)
  - "금융업 근무하려면 뭘 알아야 하나?"
  - 자본시장, 리스크 관리, 규제
- 기업별 분석 자료 정리
  - "삼성금융, 한국은행, KB금융 중 어디?"
  - 기업별 비전, 연봉, 면접 후기
- 경제 뉴스 모니터링 (시사 질문 대비)

**장기적 필요**:
- 업계 지식 장기 축적 (입사 후도 활용)
- 경제 센스 개발 (생각의 폭 확대)
- 경력 관련 네트워킹 (선배 추천, 경험 공유)

### 세컨드브레인 사용 방식

#### Monthly Workflow
```
월 초:
  1. 취업 준비 계획 수립 (면접 예상 기업 선정)
  2. 세컨드브레인에 관심 기업 등록

월중:
  1. 자동으로 수집되는 기업 뉴스 + 경제 동향
  2. 시간 날 때마다 조금씩 읽기 (15분/일)

월말:
  1. "한국은행 면접 대비 - 최근 3개월"
  2. 5분 분석 보고서 생성
  3. 내용 정리 → 면접 자료로 변환

면접 일주일 전:
  1. 최종 점검 (경제 뉴스 + 기업 분석 + 기초 지식)
  2. 예상 질문과 자료 연결
```

#### 중요한 순간
**면접 직전** (가장 중요):
- "최근 3개월 금융시장 어떻게 변했나?"
- "우리 기업 이슈는?" (ISMS 인증 취소 등)
- 통계 + 뉴스 + 기초 지식 통합

**경제 수업 또는 리포트 작성**:
- "경제학적 근거 제시"
- 축적된 자료로 학점 향상

### 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 일일 학습 시간 | 30분 (자동 수집 덕분) | 로그 |
| 지식 축적 | 3개월 100개 노트 | 통계 |
| 면접 준비율 | 기업 정보 90% 이상 | 체크리스트 |
| 면접 합격률 | 목표 기업 1곳 이상 합격 | 결과 |
| 자신감 | "준비 잘 했다"는 느낌 | 자가 평가 |

### 참여 타이밍

| Phase | 역할 | 활동 |
|-------|------|------|
| Phase B | 도메인 추천 | 경제/금융 소스 추천 |
| Phase D | 초기 사용자 | "면접 준비" 쿼리로 테스트 |
| 6개월 후 | 정기 사용자 | 월간 분석 보고서 생성 |
| 1년 후 | 입사 후 | 직무 관련 지식 축적 |

#### Project-based Workflow
```

완성 후:
  1. 완성 프로젝트 세컨드브레인에 추가 (포트폴리오용)
```

#### 중요한 순간
**클라이언트 미팅 직전** (가장 중요):
- "왜 이 색상을 선택했나?" (과학적 근거)
- "최신 트렌드에 맞춘 설계입니다" (데이터 기반)

**새 프로젝트 시작**:
- 관련 사례 + 트렌드 정리 → 협력사 디자이너와 공유

| 클라이언트 만족도 | "트렌디하다" 평가 | 설문 |

#### Monthly Workflow
```
월초:
  1. n8n이 자동으로 규제 뉴스 수집

월중:
  1. 실시간 모니터링 (자동)
  2. 관련 사안 발생 시 세컨드브레인 검색

  1. "이달 금융업 규제 변화 - 우리 회사 영향도"
```

**월간 리뷰 직전** (가장 중요):
- "이달 주요 규제는?"
- "우리 회사에 영향 있나?"
- 보고서 자동 생성 → 시간 절감

**즉시 대응 필요시**:
- "새로운 규제 공시됨"
- "과거 유사 사례는?" (세컨드브레인 검색)
- 경험 활용 → 빠른 대응

### 성공 지표 (KPI)

```

   - 사무원: "우리 회사 준비는?"

   "상반기 보수적 투자 + 규제 대응 강화"
```

## 9. 온보딩 및 교육 계획

### 각 페르소나별 온보딩

#### 개발자 (자신)
- 이미 시스템 이해 → 최소 교육
- Phase별로 자가 검증

#### 목사님
- **교육 시간**: 1시간 (한 번)
- **내용**: "셋톱박스" 처럼 사용하기
  1. 질문만 던지기 (자세한 검색 불필요)
  2. 분석 결과 읽기
  3. "유용함" 피드백하기 (선택)
- **지원**: 개발자가 설교 주제 입력 (자동화)

#### 대학생
- **교육 시간**: 30분 (온라인 가이드)
- **내용**: 면접 준비에 활용하는 법
  1. 관심 기업 등록
  2. 경제 뉴스 자동 수집 (백그라운드)
  3. 면접 1주일 전 분석 요청
- **지원**: 1:1 초기 안내

#### 디자이너
- **교육 시간**: 15분 (비디오)
- **내용**: 프로젝트마다 트렌드 조사
  1. "프로젝트명 + 2026 트렌드"로 질문
  2. 분석 결과 → 제안서에 활용
- **지원**: 이메일 가이드

#### 사무원
- **교육 시간**: 30분
- **내용**: 월간 보고서 생성
  1. 매달 첫 날 분석 요청
  2. 보고서 자동 생성
  3. 경영진에게 보고
- **지원**: 1:1 초기 안내 + 템플릿 제공

**해결**: 요약 자동화 (1~2쪽만)
