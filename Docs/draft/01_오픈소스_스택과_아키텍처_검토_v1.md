|------|------|------|

### 소셜미디어

| 도구 | 역할 | 비고 |
|------|------|------|

> **Threads 주의**: 공식 API는 자기 계정 게시물만 읽기 가능(2025 기준). 타인 피드 수집은 비공식 방법 필요.

## 2. 검색·분석 레이어

### GPT Researcher (채택)

- **역할**: 수집된 자료 기반 자동 리서치 리포트 생성
- **장점**: 로컬 문서 컨텍스트 지원, 커스텀 프롬프트, FastAPI 인터페이스
- **vs Perplexica**: GPT Researcher는 리포트 생성에 특화, 오프라인 소스 활용 가능 → 채택

```
설치: pip install gpt-researcher
용도: 주간 트렌드 요약, 논문 클러스터 리포트, 경제 지표 분석 등
```

## 5. 확장성 전략

### 자료 증가 시 대응

| 단계 | 전략 |
|------|------|
| **초기 (수천 건)** | Obsidian Dataview + SQLite로 충분 |
| **중기 (수만 건)** | **Chroma** 또는 **Qdrant** 벡터 DB 도입 → 의미 검색 가능 |
| **장기 (수십만 건)** | **GraphRAG** (Microsoft) — 엔티티·관계 그래프 기반 RAG |

### 아카이빙 전략

- Obsidian 연도별 폴더 구조 (YYYY/MM/주제)
- 원시 데이터는 SQLite에 보존, Obsidian에는 요약·링크만 저장
- 분기별 Chroma 인덱스 리빌드

### 벡터 DB 추천

| DB | 장점 | 선택 기준 |
|----|------|---------|
| **Chroma** | 로컬 설치 간단, Python 친화 | 소규모 개인 프로젝트 |
| **Qdrant** | 고성능, Docker, REST API | 팀 공유·확장 예정 시 |

## 6. 공유 전략

### 방법 1 — n8n → Notion 자동 게시

```
n8n 워크플로우:
  트리거: 매주 월요일 오전 9시
  액션 1: GPT Researcher 리포트 생성
  액션 2: Notion API로 지정 DB에 페이지 생성
  결과: 팀원이 노션에서 주간 리포트 열람
```

- 장점: 별도 인프라 불필요, 노션 공유 권한 활용
- 단점: 노션 구독 필요, 실시간 업데이트 불편

### 방법 2 — Quartz 정적 웹사이트

```
Obsidian Vault → Quartz 빌드 → GitHub Pages 배포
  - 공개 노트만 선별 publish: true 태그
  - 검색 기능 내장 (Fuse.js)
  - 그래프 뷰로 노트 연결 시각화
```

- 장점: 완전 무료, 외부 공유 가능, 커스터마이징 자유
- 단점: 정적이므로 업데이트마다 빌드 필요

### 추천 조합

> **일상 공유**: n8n → Notion (팀 내부)
> **외부 공개**: Quartz → GitHub Pages (선택적)
